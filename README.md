Project K.A.R.T.H.I.K. (Kinetic Algorithmic Real-Time High-Intensity Knight)An institutional-grade, sub-millisecond algorithmic trading microservices architecture designed for regime-adaptive alpha generation and sovereign risk management.📐 Architecture OverviewProject K.A.R.T.H.I.K. is built as a distributed, polyglot pipeline to bypass the Python Global Interpreter Lock (GIL) and achieve institutional-grade execution speeds.Data Ingestion Layer (C++): Utilizes uWebSockets and simdjson for ultra-low latency tick parsing, normalized into Protobuf binary format.Intelligence Layer (Rust & Python): Heavy microstructure math (VPIN, Greeks, OFI) is offloaded to a Rust-based tick engine via PyO3 for microsecond-level computation.Orchestration (Python): High-level strategy logic, regime detection, and risk daemons are implemented in asynchronous Python for rapid iteration and flexibility.Communication (ZMQ & SHM): Uses ZeroMQ for high-speed binary messaging (~40µs) and zero-copy Shared Memory (SHM) for market state vectors ($<1$µs latency).🧩 Core Microservices1. Market Sensor (daemons/market_sensor.py)The "Eyes of the System" that transforms raw ticks into a Composite Alpha Score ($S_{total}$).Computes Hurst Exponent for regime detection (Trending vs. Mean-Reverting).Calculates Order Flow Imbalance (OFI) and VPIN for flow toxicity monitoring.Uses an Adaptive SuperTrend Oscillator (ASTO) to normalize price volatility.2. Strategy Engine (daemons/strategy_engine.py)A modular engine hosting a suite of regime-aware strategies:TastyTrade 0DTE: High-frequency theta harvesting for NIFTY, BANKNIFTY, and SENSEX expiries using Iron Butterflies.Iron Condor: Neutral market strategy with dynamic delta-based strike selection and "Hedge Hybrid" futures integration.Gamma Scalping: Real-time delta hedging to capture realized volatility moves.Kinetic Hunter: Momentum Specialist triggered by smart flow alignment and Whale Pivot confirmation.3. Liquidation Daemon (daemons/liquidation_daemon.py)The sovereign risk guardian enforcing a Triple-Barrier exit system:Horizontal Barriers: ATR-based Take-Profit and Stop-Loss levels.Vertical Barrier: Optimal stopping logic to kill stagnant trades (e.g., 150-300s window for 0DTE).Safety Gates: Monitors slippage budgets and cross-asset drawdown limits to trigger global square-offs.4. Unified Execution Bridge (daemons/unified_bridge.py)A multiplexer providing high-fidelity execution across three realities:Shadow: A counterfactual ledger for slippage and efficacy analysis.Paper: Simulated fills using regime-aware slippage models.Actual: Live strike gateway integrated with the Shoonya (Noren) API.5. Order Reconciler (daemons/order_reconciler.py)A specialized "Janitor" daemon that ensures multi-leg basket integrity:Monitors in-flight orders for partial fills or broker rejections.Triggers automated Circuit Breaker Rollbacks for broken spreads to prevent unhedged exposure.🛠️ Technology StackLanguages: Python 3.11+, C++20, Rust.Messaging: ZeroMQ (ZMQ), Protobuf.Data Stores: TimescaleDB (trade logs/PnL), Redis (shared state/LUA scripts).Backend: FastAPI, uvicorn.Frontend: React (Mobile-First Dashboard).Infrastructure: Docker, Docker Compose, Google Cloud Compute.🚀 Getting StartedLocal SetupConfigure environment variables by copying .env.example to .env.Initialize the infrastructure using the provided Makefile:Bashmake build
